# Capabilities — {{projectName}}

The single place that tracks the **skills** and **MCP servers** installed in this
project — what each is for, and the policy for choosing among them. **Agents: read
this whenever a task might reach for a skill or an MCP server.** The inventory tables
near the bottom are regenerated by `neptr index`; the policy above them is yours to
curate by hand.

## Usage policy

- **A capability must be listed here to be used.** If a skill or MCP server isn't in
  the tables below, don't invoke it. Install it first so it's tracked
  (`neptr skill "<keywords>"` / `neptr mcp "<keywords>"`), or leave it alone.
- **Prefer the fewest tools that solve the task.** Plain code and the standard toolset
  are the default; reach for a skill or MCP server only when it clearly fits. Every
  extra tool in play is another way for things to conflict.
- **One capability, one tool.** When two entries overlap (two browser skills, two
  database MCP servers, two "code review" skills), pick one for the task and stick with
  it. Record the winner in the precedence list below so the next agent doesn't
  re-litigate it.
- **Don't install a new capability if an existing one already covers it.** Check the
  tables first. Only add a new skill or MCP server when nothing installed does the job.
- **This file wins on conflicts.** If a skill's guidance contradicts the constitution
  ([CONSTITUTION.md](CONSTITUTION.md)) or these rules, stop and ask — don't silently
  follow the skill.
